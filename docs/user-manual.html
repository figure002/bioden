Using bioden ¶

Bioden (bioden.py) is a command-line interface (CLI) application. This means that you can run it from the command line (the "terminal" on UNIX systems or "DOS" on Windows systems). Bioden also comes with a graphical frontend (bioden-gtk.pyw), which is probably more suitable for Windows users. Skip to the GUI section for help on using the frontend, or go to the CLI section for help on running bioden from the command-line.

Using bioden (GUI) ¶

Start bioden by running bioden-gtk.pyw. This can be done by (double) clicking it. Note that bioden.py and bioden-gtk.pyw must be in the same folder.

Once bioden is started, you'll be presented with its main window. The window has the following elements:

Select data file
The CSV file containing the biomass and density data to be normalized.
Select output folder
Select the folder to which all output files should be saved.
Property for calculations
The property to perform the calculations on. Can be either 'biomass' or 'density'.
Advanced Options
Clicking this toggle button shows/hides the advanced options.
Field delimiter of the input file:
A one-character string used to separate fields in the input CSV file. It defaults to a semicolon (;). By default Excel uses the semicolon to export CSV file. If the CSV file is exported by a different application, make sure this character matches the delimiter in the data file. You can check this simply by opening the data file in a text editor (like Notepad).
Quote character of the input file:
A one-character string used to separate fields in the input CSV file. It defaults to a double quote (").
Round values to n decimals:
Number of decimals to round values to in the output files. Value "-1" (default) means do not round.
Process Data
Clicking the button starts the data processing. While it's processing the data, a progress dialog is displayed. Be patient, the calculations could take some time to finish based on the amount of data in the data file.

Using bioden (CLI) ¶

Start your terminal application and move to the folder containing bioden.py. Then type the following command to execute bioden: ./bioden.py

Running bioden like that (without arguments) will display its usage information in the terminal. 

serrano@saibot:~$ ./bioden.py
bioden 0.1

Usage: bioden.py [options] -i [input_file] -o [output_folder]

Options:
   -h,--help                     Show this usage information.
   -i [input_file]               Input CSV file (*.csv).
   -o [output_folder]            Folder to save the output files in.
   -d,--delimiter "[char]"       A one-character string used to
                                 separate fields. It defaults to ';'.
   -q,--quotechar "[char]"       A one-character string used to quote
                                 fields containing special characters.
                                 It defaults to '"'.
   -p,--property [property]      The property on which to perform the
                                 calculations. Can be either 'biomass'
                                 (default) or 'density'.
   -r,--round [n]                Round values to [n] decimals. Default
                                 is not to round.
The usage information should be straightforward. Here is a simple usage example:
./bioden.py -i input.csv -o data/ -p biomass

Output ¶

Bioden creates different files in the specified output folder.

Transponed data files (transponed_<property>_<ecotope>.csv)
These files contain the raw data, but the data has transponed (=moved). The first column in the file shows a list of all taxa. Each row is a specific sample indicated with the sample code. Each data row contains the biomass/density for that taxa and sample code.
