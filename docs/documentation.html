Bioden Documentation

Welcome to the documentation for Bioden.

Introduction ¶

Bioden is a data normalizer and transponer for CSV files containing taxon biomass/density data for ecotopes.

Requirements ¶

Bioden has the following third-party dependencies:

  • Python
  • GTK+
  • PyGTK
  • PyCairo
  • PyGObject

The Windows® installer comes bundled with these dependencies. The third party tools bundled with Bioden are property of their individual authors and are governed by their individual applicable license.

Installing Bioden ¶

Microsoft® Windows®

For Windows users, an installer is available that includes the Bioden package together with necessary pre-requisites listed under Requirements.

GNU/Linux

GNU/Linux users can use the source package for Bioden. The source package doesn't contain the listed pre-requisites though. You'll have to install missing requirements manually with your package manager.

Using Bioden ¶

On Windows, go to Start menu > All Programs > GiMaRIS > bioden > bioden.

Linux users can just run bioden.pyw after unpacking the archive.

Once bioden is started, you'll be presented with its main window. The main window has the following components:

Select input data file
The CSV file containing the biomass and density data to be processed. This file must be in a specific format. This format is described in the Input File Format section.
Select output folder
Select the folder to which all output files should be saved. The output file are described in the Output Files section.
Property for calculations
The property to perform the calculations on. Can be either 'biomass' or 'density'. Note that the input data file must have the specified property stored.
Advanced Options
Clicking this toggle button shows/hides the advanced options.
Field delimiter of the input file:
A one-character string used to separate fields in the input CSV file. It defaults to a semicolon (;). By default Excel uses the semicolon for CSV files. If the CSV file is exported by a different application, make sure this character matches the delimiter in the data file. You can check this by opening the data file in a text editor (like Notepad).
Quote character of the input file:
A one-character string used to separate fields in the input CSV file. It defaults to a double quote (").
Round values to n decimals:
Number of decimals to round values in the output files to. Value "-1" (default) means do not round.
Process Data
Clicking this button starts the data processing. While it's processing the data, a progress dialog is displayed. Be patient, the calculations could take some time to finish based on the amount of data in the data file.

Input File Format ¶

The input data file must be a CSV file in a specific format. The default settings for Bioden have been set for the format in which Excel creates CSV files. This means that by default Bioden expects a semicolon (;) as a field separator. The quote character defaults to a double quote ("). The Advanced Options in Bioden allow you to change these defaults, should your input CSV file use a different field delimiter or quotechar.

Except for the CSV format, Bioden also expects a format in which the data has been stored in the CSV file. Bioden expects the following content format:

  1. The first line in the CSV file must be the header. The header contains the column names. Bioden expects at least the following six columns: "compiled ecotope", "sample code", "standardised taxon", "density", "biomass", and "sample surface".
  2. Bioden isn't very strict with the column names. The column names aren't case sensitive. So a column with name "BiOmAsS" works just as good. Also Bioden uses any column that contains any of the expected column names. So a column with name "Sum Of Density" would be used as the "density" column, because it contains the string "density".
  3. Bioden ignores the columns that don't match the expected column names. So it's OK if the input file contains other columns.
  4. The header line must be followed by the data lines.
  5. Bioden automatically corrects numbers that have been stored in non-English format. This means that commas are replaced by dots (e.g. 12,5 will be converted to 12.5).

Viewing Output Files ¶

The output files created by Bioden are stored in CSV (Comma Separated Values) format. You can view these output files in any spreadsheet application (e.g. Microsoft® Excel®, OpenOffice.org Calc). Your spreadsheet application however does need to know some basic information about the CSV file before it can view it properly. For example, it needs to know the character used as a field delimiter, which is a comma. So you need to tell your spreadsheet application to use the comma character as the field delimiter, along some other properties. I will explain how to do this for the two most popular spreadsheet applications.

Microsoft® Excel® 2007

  1. Start Microsoft® Excel®. Make sure you have open the Excel® 2007 worksheet that you would like to import the ouput data file into.
  2. Go to the "Data/Gegevens" tab.
  3. Click on the "From Text/Van text" button. This will open the file select browser window. Navigate to where the output files are stored. Then select one of the output files. All output files have the extension ".csv".
  4. After selecting an output file, the "Text Import Wizard/Wizard Text importeren" window is displayed. In that window set:
    • "Original Data Type/Oorspronkelijk gegevenstype" to "Delimited/Gescheiden"
    • "File origin/Oorspronkelijk bestand" to "United States/Verenigde Staten"
    Click the "Next/Volgende" button.
  5. In the next window that appears we need to change the Delimiter from Tab to Comma. Uncheck "Tab" and check "Comma/Komma" under "Delimiters/Scheidingstekens".
  6. Press "Finish/Voltooien". The data from the output file should now be displayed in the correct columns.

OpenOffice.org Calc

  1. Go to the folder containing the output files.
  2. Right-click on an output file, and select "Open With > OpenOffice.org Calc". The "Text Import" window should now appear.
  3. In the Text Import window, set:
    • "Language" to "English (USA)"
    • "Separator options" to "Separated by" to "Comma"
  4. Press OK. The data from the output file should now be displayed in the correct columns.

Troubleshooting ¶

"This application has failed to start because the application configuration is incorrect. Reinstalling the application my fix this problem."
This error is most likely caused by a missing DLL. The installer for Bioden has the option to install the required DLL. To fix this problem, run the installer for Bioden and make sure that the "Microsoft Visual C++" option is checked. This will install the required DLL for Bioden.
